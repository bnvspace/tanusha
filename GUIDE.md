## 1. Роли пользователей

В системе три роли:

| Роль | Что может делать |
|---|---|
| **Студент** | Просматривать материалы, сдавать задания, проходить тесты, видеть свои оценки |
| **Преподаватель** | Всё что студент + добавлять/редактировать материалы, задания, тесты, проверять работы, видеть статистику |
| **Администратор** | Всё что преподаватель + управление учётными записями пользователей (блокировка, создание) |

**Стандартные учётные данные:**

| Роль | Логин | Пароль |
|---|---|---|
| Администратор | `admin` | `admin123` |
| Преподаватель | `teacher` | `teacher123` |
| Студент (тест) | `student1` | `student123` |

## 2. Главная страница — вход и регистрация

**URL:** http://localhost:5000

Страница состоит из двух частей:
- **Левая часть** — описание курса, его возможности
- **Правая часть** — форма входа

### Вход в систему
1. Введите **Логин** (не email, именно username)
2. Введите **Пароль**
3. Нажмите **«Войти →»**
4. После входа:
   - Администратор/Преподаватель → попадают в **Панель управления**
   - Студент → попадает в **Главную страницу курса**

### Регистрация студента
1. Нажмите ссылку **«Зарегистрироваться»** под кнопкой входа
2. Заполните форму:
   - **ФИО** — полное имя (напр. *Иванов Иван Иванович*)
   - **Логин** — уникальный, только латиница (напр. *ivanov*)
   - **Email** — уникальный email
   - **Пароль** — любой, запомните его
3. Нажмите **«Зарегистрироваться»**
4. После регистрации вернётесь на страницу входа — войдите с новыми данными

> **Примечание:** Администратор также может создавать учётные записи вручную через раздел «Пользователи».

## 3. Панель администратора

**URL:** http://localhost:5000/admin/  
**Доступ:** Преподаватель, Администратор

После входа администратор/преподаватель видит **дашборд** с четырьмя карточками статистики:
- **Студентов** — сколько зарегистрировано студентов
- **Ждут проверки** — число сданных, но ещё не проверенных работ
- **Всего ответов** — общее количество сданных работ
- **Тестов пройдено** — сколько раз тесты были завершены

Ниже находится блок **«Быстрые действия»**:
- 🔴 **Проверить задания** — ведёт к списку ожидающих проверки работ
- **Добавить задание** — форма создания нового задания
- **Добавить материал** — форма загрузки материала
- **Создать тест** — конструктор теста
- **Статистика** — аналитика курса

Справа — список последних студентов.

**Боковое меню (сайдбар):**
- 📊 **Панель управления** — главный дашборд
- 👥 **Студенты** — список всех студентов
- ✏️ **Проверка заданий** — очередь работ
- 📚 **Управление курсом** — структура курса
- 📈 **Статистика** — аналитика
- 🔑 **Пользователи** — управление аккаунтами (только для Администратора)

## 4. Управление пользователями

**URL:** http://localhost:5000/admin/users  
**Доступ:** Только Администратор

### Создание нового пользователя
Используется, когда администратор сам создаёт аккаунт для студента.

1. В верхней части страницы заполните форму:
   - **ФИО** — полное имя студента
   - **Логин** — уникальный логин
   - **Email** — уникальный email
   - **Пароль** — начальный пароль, который вы сообщаете студенту
   - **Роль** — выберите `Студент`, `Преподаватель` или `Администратор`
2. Нажмите **«Создать»**

### Блокировка / разблокировка
В таблице всех пользователей у каждой строки есть кнопка:
- **«Заблокировать»** — лишает пользователя возможности войти (кнопка красная)
- **«Разблокировать»** — восстанавливает доступ (кнопка зелёная)

> Себя заблокировать нельзя — у вашей строки кнопка отсутствует.

## 5. Управление курсом и неделями

**URL:** http://localhost:5000/admin/course  
**Доступ:** Преподаватель, Администратор

### Редактирование информации о курсе
В верхней карточке можно изменить:
- **Название курса** — отображается у студентов в дашборде и шапке
- **Описание** — общее описание курса
- **Цели** — цели обучения (показываются на главной студента)
- **Задачи** — конкретные задачи курса (виде маркированного списка)
- **Содержание обучения** — общая информация о структуре

Нажмите **«💾 Сохранить»**.

### Структура курса — Недели
Курс разбит на **недели**. Каждая неделя — это тематический модуль.

**Добавить неделю:**
1. Прокрутите страницу вниз до блока «➕ Добавить неделю»
2. Введите название (напр. *Неделя 5: Технические переговоры*)
3. Нажмите **«Добавить»**
4. Неделя появится в списке выше

**Удалить неделю:**
- В правом углу заголовка недели нажмите **«Удалить неделю»**
- Подтвердите в диалоге
- ⚠️ Удаляются **все материалы, задания и тесты** этой недели!

В каждом блоке недели отображаются вложенные материалы, задания и тесты с кнопками редактирования (✏️) и удаления (🗑).

## 6. Добавление учебных материалов

**URL:** http://localhost:5000/admin/material/add  
(или кнопка «+ Материал» на странице курса / «Добавить материал» в дашборде)  
**Доступ:** Преподаватель, Администратор

### Типы материалов

| Тип | Когда использовать |
|---|---|
| 📝 **Текст** | Правила, объяснения, конспекты прямо на странице |
| 📄 **Файл** | PDF, Word, PPT, ZIP и другие документы для скачивания |
| 🎬 **Видео** | Ссылка на YouTube или другой видеохостинг |
| 🎵 **Аудио** | Аудиофайл (MP3) для прослушивания |
| 🔗 **Ссылка** | Ссылка на внешний сайт или ресурс |
| 🎯 **Интерактивное** | Интерактивное задание (файл с упражнением) |

### Как добавить материал

1. Выберите **Неделю**, к которой относится материал
2. Выберите **Тип материала** — форма автоматически покажет/скроет нужные поля:
   - Для *Видео* и *Ссылки* — появится поле **URL**
   - Для *Файла*, *Аудио*, *Интерактивного* — появится поле загрузки **файла**
3. Введите **Название** материала (напр. *Лексика по теме «Теплообмен»*)
4. При необходимости добавьте **Содержание/описание** — текст, который студент увидит на странице
5. Укажите **Дату открытия** — если нужно скрыть материал до определённого момента (напр. материал откроется в день занятия). Оставьте пустым для немедленного доступа
6. Выберите **Видимость**: *Видимо студентам* или *Скрыто*
7. Нажмите **«✅ Добавить материал»**

### Редактирование материала
На странице курса рядом с каждым материалом есть кнопка **✏️** — откроет форму редактирования с теми же полями.

### Ограничения файлов
- Допустимые форматы: PDF, DOC, DOCX, TXT, PPT, PPTX, XLS, XLSX, MP3, MP4, PNG, JPG, JPEG, ZIP
- Максимальный размер: **50 МБ**

## 7. Добавление заданий

**URL:** http://localhost:5000/admin/assignment/add  
**Доступ:** Преподаватель, Администратор

Задание — это работа, которую студент должен выполнить и сдать (текстом или файлом). Преподаватель затем проверяет и выставляет оценку.

### Как добавить задание

1. Выберите **Неделю**
2. Выберите **Видимость** (Видимо / Скрыто)
3. Введите **Название задания** (напр. *Перевод технического текста — Неделя 2*)
4. Введите **Описание задания** — подробно опишите:
   - Что нужно сделать
   - Требования к выполнению
   - Критерии оценивания
   - Объём работы
5. Укажите **Дедлайн** — дата и время, до которого принимаются работы. После дедлайна форма сдачи у студента исчезает. Оставьте пустым если дедлайна нет
6. Укажите **Дату открытия** — когда задание станет видно студентам
7. Нажмите **«✅ Добавить задание»**

### Статусы задания (у студента)
- ⏳ **Ожидает проверки** — студент сдал, преподаватель ещё не смотрел
- ✅ **Проверено** — преподаватель выставил оценку
- 🔄 **Требует доработки** — работа возвращена с комментарием

## 8. Создание тестов

**URL:** http://localhost:5000/admin/test/add  
**Доступ:** Преподаватель, Администратор

Тест — задание с автоматической проверкой. Студент отвечает на вопросы, система сразу подсчитывает результат.

### Настройки теста

1. Выберите **Неделю**
2. Укажите **Лимит времени** (в минутах). Например: `30`. Если `0` — время не ограничено
3. Введите **Название теста** (напр. *Лексический тест №1*)
4. Добавьте **Описание** — инструкции для студента
5. Выберите **Видимость**
6. **Показывать ответы** — если *Да*, студент после прохождения увидит правильные/неправильные ответы. Если *Нет* — только итоговый балл
7. Укажите **Дату открытия** при необходимости

### Добавление вопросов

Нажмите **«+ Добавить вопрос»** — появится блок вопроса.

#### Три типа вопросов:

**1. Один правильный ответ (single)**
- Студент видит группу радиокнопок, может выбрать только один вариант
- В конструкторе: добавьте варианты кнопкой «+ Добавить вариант», **отметьте галочкой** правильный

**2. Несколько правильных ответов (multiple)**
- Студент видит чекбоксы, может отметить несколько
- Правильным считается только точное совпадение всех отмеченных вариантов
- В конструкторе: отметьте галочками **все** правильные варианты

**3. Ввод текста (text)**
- Студент вводит ответ вручную
- Проверка — точное совпадение (без учёта регистра)
- В конструкторе: в специальное поле введите правильные ответы через запятую (напр. `boiler, котёл`)

#### Пример добавления вопроса
```
Как называется устройство для нагрева воды?
Тип: Один правильный ответ
Варианты:
[✓] Котёл
[ ] Турбина
[ ] Конденсатор
[ ] Насос
```

Можно добавить любое количество вопросов. Чтобы удалить вопрос — нажмите **✕ Удалить** в его заголовке.

8. Нажмите **«✅ Создать тест»**

> **Оценка за тест** = количество правильных ответов / всего вопросов × 100%

## 9. Проверка работ студентов

**URL:** http://localhost:5000/admin/review  
**Доступ:** Преподаватель, Администратор

Страница разделена на две части:

### Верхняя часть — «Ожидают проверки»
Список всех сданных работ со статусом *«ожидает проверки»* (отсортирован по дате сдачи — сначала самые старые).

Для каждой работы показано:
- Имя студента
- Название задания и неделя
- Дата и время сдачи
- Кнопка **📄 Файл** (если студент приложил файл) — открывает файл в новой вкладке

### Как проверить работу

1. Нажмите кнопку **«Проверить»** рядом с нужной работой
2. Откроется страница с двумя колонками:
   - **Слева** — текст задания и ответ студента (текст + ссылка на файл)
   - **Справа** — форма оценивания
3. Заполните форму:
   - **Оценка (0–100)** — числовая оценка
   - **Статус:**
     - ✅ *Проверено* — работа принята
     - 🔄 *Требует доработки* — нужна переработка, студент получит уведомление
   - **Комментарий** — напишите развёрнутый отзыв (виден студенту на странице задания)
4. Нажмите **«✅ Подтвердить оценку»**

> Оценку можно изменить в любой момент, повторно открыв ту же работу.

### Нижняя часть — «Последние проверенные»
История последних 50 проверенных работ для справки.

## 10. Статистика

**URL:** http://localhost:5000/admin/statistics  
**Доступ:** Преподаватель, Администратор

Страница показывает:

**Карточки вверху:**
- Количество студентов
- Всего ответов на задания
- Проверено ответов
- Пройдено тестов

**График прогресса проверки:**
Прогресс-бар, показывающий какой % работ уже проверен.

**Распределение оценок:**
Горизонтальные прогресс-бары, разбивающие оценки на диапазоны:
- 81–100 (отлично)
- 61–80 (хорошо)
- 41–60 (удовлетворительно)
- 21–40 (слабо)
- 0–20 (неудовлетворительно)

## 11. Кабинет студента

После входа студент видит меню в сайдбаре:
- 🏠 **Главная** — обзор курса
- 📚 **Учебные материалы** — все материалы по неделям
- 📝 **Задания** — список заданий и их статусы
- 🧪 **Тесты** — список доступных тестов
- 🏆 **Мои оценки** — успеваемость и прогресс

### Главная страница курса
Показывает:
- Карточки: кол-во недель, выполненных заданий, пройденных тестов
- Описание курса, цели, задачи, содержание обучения
- Структуру курса — список недель с количеством материалов

### Учебные материалы
Все материалы сгруппированы по неделям (аккордеон — клик раскрывает неделю).

Для каждого материала отображается:
- Иконка типа (📄 файл, 🎬 видео, 🎵 аудио, 🔗 ссылка, 📝 текст)
- Название и краткое описание
- Ссылка для скачивания/просмотра
- Ссылки на задания и тесты этой недели

## 12. Как студент сдаёт задание

1. Нажмите **«Задания»** в сайдбаре
2. Выберите задание из списка, нажмите **«Открыть задание»**
3. Прочитайте условие задания
4. Заполните поле **«Ваш ответ»** — можете ввести текст и/или прикрепить файл
5. Нажмите **«📤 Отправить ответ»**

**После сдачи** статус изменится на ⏳ *«Ожидает проверки»*

**Если получили статус 🔄 «Требует доработки»:**
- Прочитайте комментарий преподавателя
- Переработайте ответ и нажмите **«Отправить исправленную версию»** — статус снова станет «Ожидает проверки»

> Пока работа не получила статус ✅, её можно редактировать и пересдавать.

---

## 13. Как студент проходит тест

1. Нажмите **«Тесты»** в сайдбаре
2. Выберите тест из списка, нажмите **«Начать тест»**
3. Ответьте на все вопросы:
   - **Один вариант** — кликните на нужный кружок
   - **Несколько вариантов** — отметьте все подходящие чекбоксы
   - **Текстовый ввод** — введите ответ в поле
4. Если установлен таймер — следите за обратным отсчётом вверху страницы. При истечении времени тест отправится автоматически
5. Нажмите **«✅ Завершить тест»**

**Страница результатов** покажет:
- Итоговый балл (например: *7 из 10 — 70%*)
- Цветную индикацию: 🟢 ≥75%, 🟡 ≥50%, 🔴 <50%
- Если преподаватель разрешил — детальный разбор каждого вопроса:
  - ✅ Правильно
  - ❌ Неправильно (виден правильный ответ)

> Тест можно пройти **только один раз**. После завершения кнопка «Начать» заменяется на «Посмотреть результат».

## 14. Оценки и прогресс

**URL:** http://localhost:5000/grades  
**Доступ:** Студент

Страница «Мои оценки» показывает:

**Карточки вверху:**
- Средняя оценка за задания (0–100)
- Количество проверенных заданий
- Количество пройденных тестов

**Прогресс-бар** — общий прогресс сдачи всех проверяемых работ.

**Таблица успеваемости** — все задания и тесты с колонками:
- Тип (задание / тест) и неделя
- Название
- Статус
- Оценка в % (с цветовой индикацией)
- Комментарий преподавателя

## 🚀 Рекомендуемый порядок настройки нового курса

```
1. Войти как администратор или преподаватель
2. → Управление курсом → отредактировать название и описание курса
3. → Управление курсом → добавить нужные недели (напр. 8 недель)
4. Для каждой недели:
   a. Добавить материал (+)
   b. Добавить задание (+)
   c. Создать тест (+)
5. → Пользователи → создать аккаунты студентов (или они сами регистрируются)
6. Студенты входят, изучают материалы, сдают задания, проходят тесты
7. → Проверка заданий → регулярно проверять поступившие работы
8. → Статистика → следить за прогрессом группы
```
