# Образовательный портал на PHP

Проект переведён на `PHP + SQLite`.

## Быстрый старт

```bash
php -S localhost:8000
```

Откройте в браузере: `http://localhost:8000/index.php`

Если у вас Apache или Nginx, достаточно направить Document Root на папку проекта.

## Что происходит при первом запуске

- `config.php` автоматически подключает `bootstrap.php`
- если `storage/portal.db` отсутствует, таблицы создаются автоматически
- если в корне проекта есть legacy-`portal.db`, приложение один раз копирует её в `storage/portal.db`
- если в базе нет стартовых данных, создаются курс, недели и стандартные пользователи
- если в базе лежат старые Flask/Werkzeug-хеши для `admin`, `teacher`, `student1`, они один раз переводятся в формат PHP

Это значит, что ручную миграцию не нужно делать после каждого коммита. По сути она нужна только для старой базы и теперь встроена в инициализацию приложения.

## Учётные записи по умолчанию

| Роль | Логин | Пароль |
|---|---|---|
| Администратор | `admin` | `admin123` |
| Преподаватель | `teacher` | `teacher123` |
| Студент | `student1` | `student123` |

## Структура проекта

```text
tanusha/
├── bootstrap.php        # создание схемы SQLite и стартовых данных
├── config.php           # подключение к БД и запуск bootstrap
├── auth.php             # сессии и авторизация
├── index.php            # главный роутер PHP-приложения
├── pages/               # PHP-страницы портала
├── static/              # стили и статика
├── uploads/             # загружаемые файлы
├── storage/             # рабочая SQLite база вне git-потока
├── lang.php             # переводы
└── portal.db            # legacy SQLite база для одноразового переноса
```

## Примечание по legacy-данным

В репозитории ещё могут оставаться старые Python/Flask-файлы как технический хвост прошлой версии, но рабочий поток проекта теперь должен идти через `index.php` и `PHP`-страницы.
